SQL 基本语法操作

SQL简介

SQL 是一门 ANSI 的标准计算机语言，用来访问和操作数据库系统。SQL 语句用于取回和更新数据库中的数据。

RDBMS

  • RDBMS指的是关系型数据库管理系统
  • 它是SQL的基础，也是所有现代数据库的基础，常见的有MS SQL Server, IBM DB2, Oracle, MySQL 以及 Microsoft Access。

SQL 语言的分类

  • 数据操作语言 (DML)
    • UPDATE - 更新数据库表中的数据
    • DELETE - 从数据库表中删除数据
    • INSERT INTO 向数据库表中插入数据
  • 数据定义语言 (DDL)
    • CREATE TABLE/VIEW/INDEX/SYN/CLUSTER
  • 数据库查询语言 (DQL)
    • SELECT <字段名表>
      FROM <表或视图名>
      WHERE <查询条件>
  • 数据库控制语言 (DCL)
    • GRANT：授权
    • ROLLBACK [WORK] TO [SAVEPOINT]:回滚

SQL 单表查询

条件查询

    select 字段 from tablename where 字段 (not)in (元素1,2,3)；

...  where 条件表达式1 and 条件表达式2；

...  where 条件表达式1 or 条件表达式2；

范围查询

select 字段1，字段2  from tablename where 字段 (not) between x1 and x2

select 字段1，字段2  from tablename where 字段 (not) in (x1,x2,...,xn)

去重查询

select distinct 字段 from tablename

模糊查询

 select 字段 from tablename  where 字段 like 条件

% 代替任意str
_ 代替一个char
[charlist] : [a,b,c] 字符中含有a or b or c
[^charlist] : 与上面对应

排序

select 字段1，字段2 from tablename order by 属性名[ASC默认|DESC]

带有函数的查询

  • AVG (平均)

  • COUNT (计数)

  • MAX (最大值)

  • MIN (最小值)

  • SUM (总合)

    例：

    select MAX(字段) from tablename

分组查询

  group by 
group_concat(字段名)//只是将符合条件的字段名拼接在一起


查找每个部门有多少名在职员工

SELECT
      dept_no as 部门,
      count( emp_no) as 人数
    FROM
      dept_emp 
    WHERE
      to_date = '9999-01-01' 
    GROUP BY
      dept_no

having 相当于 对group by 的对象筛选(功能类似于where)

每个部门人数都有了，那如果我们想要进一步知道员工人数大于30000的部门是哪些，这个时候就得用到HAVING了

SELECT
      dept_no as 部门,
      count(emp_no) as 人数
    FROM
      dept_emp 
    WHERE
      to_date = '9999-01-01' 
    GROUP BY
      dept_no
    HAVING
        count(emp_no)>30000

表连接

关于inner join， left join，right join：

举例如下：

表A记录如下：

aID aNum
1 a20050111
2 a20050112
3 a20050113
4 a20050114
5 a20050115

表B记录如下:

bID bName
1 2006032401
2 2006032402
3 2006032403
4 2006032404
8 2006032408

1.inner join

select * from a inner join B on a.aID=b.bID;

结果如下：

aID aNum biD bName
1 a20050111 1 2006032401
2 a20050112 2 2006032402
3 a20050113 3 2006032403
4 a20050114 4 2006032404

结果说明:
很明显,这里只显示出了 A.aID = B.bID的记录.这说明inner join并不以谁为基础,它只显示符合条件的记录.

2.left join

select * from A left join B on A.aID = B.bID

结果如下:

aID aNum biD bName
1 a20050111 1 2006032401
2 a20050112 2 2006032402
3 a20050113 3 2006032403
4 a20050114 4 2006032404
5 a20050115 NULL NULL

结果说明:
left join是以A表的记录为基础的,A可以看成左表,B可以看成右表,left join是以左表为准的.
换句话说,左表(A)的记录将会全部表示出来,而右表(B)只会显示符合搜索条件的记录(例子中为: A.aID = B.bID).
B表记录不足的地方均为NULL.

注：right join和left join的结果刚好相反,这次是以右表(B)为基础的,A表不足的地方用NULL填充.
